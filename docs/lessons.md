1. 存档读取使用 `eval()` 解析坐标有安全风险，应考虑使用正则或 JSON 格式。
2. 固定随机种子时需同时设置 `random.seed()` 和 `np.random.seed()`。
3. 细胞繁殖时需使用预占位机制 (`pending_positions`) 避免同一帧多个细胞落到同一位置。
4. NTs矩阵存档使用 `.npy` 格式保存，读取时需兼容旧存档（可能无NTs文件）。
5. Pygame渲染时使用 `pygame.DOUBLEBUF` 和 vsync 可减少画面撕裂。
6. 这类带随机行为的重构必须先固定日志基线，再拆模块，否则很难判断是结构变化还是行为漂移。
7. 无头回归执行时需要显式设置 `PYTHONPATH` 指向工作区根目录，否则直接运行脚本可能无法导入本地包。
8. 交互面板把空格键优先当作输入字符会吞掉全局暂停热键；应在输入为空时优先处理暂停/单步热键，再进入文本输入分支。
9. `spawn` 参数解析若用 `fields.pop('x')/pop('y')` 强制取值会触发 `KeyError('x')`；应将 `x/y` 设为可选并回落到当前世界中心坐标。
10. 存档格式新增字段（如 `direction`）后需保留旧格式回退解析，否则历史存档读取会失败。
