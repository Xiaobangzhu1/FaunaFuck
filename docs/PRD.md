# PRD - FaunaFuck 产品需求文档（高层）

## 1. 文档定位

- 本文档仅定义：项目做什么、为什么做、用户看到什么、成功标准。
- 细节规则主权在系统设计文档；若与 PRD 冲突，先标记 `待确认`，不得直接按任一方实现。
- 实施顺序与任务拆解由 `docs/IMPLEMENTATION_PLAN.md` 负责。

## 2. 项目概述

FaunaFuck 是一个类 Brainfuck 的细胞自动机模拟器。系统通过 DNA→RNA 转录驱动细胞行为，支持在生态箱中观察群体动态、规则干预与运行反馈。

## 3. 目标

- 提供可交互的生态箱运行环境（投放、运行控制、规则调整、日志反馈）。
- 支持基于 DNA/RNA 规则的细胞行为演化观察。
- 保持主要交互流程可理解、可验证、可复现。

## 4. 范围

### 4.1 In Scope（高层功能范围）

- 空生态箱启动与交互控制面板。
- 细胞投放与参数化配置能力。
- 运行控制（暂停、前进、回退、重置）。
- 运行时存档与读档控制能力。
- 日志导出与运行反馈可视化。
- DNA 规则说明浮窗与 MODE2-only 规则体系。

### 4.2 Out of Scope（非目标）

- 本 PRD 不定义逐条指令映射、逐字段状态迁移和渲染算法细节。
- 本 PRD 不承载任务排期、里程碑分工、测试执行步骤。

## 5. 核心体验与核心规则（高层）

- 用户可以在运行中观察细胞行为变化，并通过输入命令干预系统。
- 用户可以通过配置影响行为阈值与运行表现。
- 新细胞生成时应默认优先尝试父细胞朝向方向。
- 复制指令执行时应支持概率失败；失败时跳过该复制指令。
- 控制台应支持选中历史指令并复制，以及在输入框内通过 `Left/Right` 移动插入点。
- 系统需在保证可交互的前提下，提供稳定反馈与最小可复现能力。
- DNA 转录规则仅支持 MODE2；MODE1 废弃。

## 6. 错误处理（高层）

- 输入非法参数时应阻断执行并反馈错误。
- 日志调取、存档读写、浮窗交互失败时应可提示并记录，不应导致主循环不可用。
- 文档规则冲突必须先标 `待确认`，再进入实现阶段。

## 7. 验收标准（高层）

- 可完成生态箱基础交互闭环：投放、运行控制、反馈查看。
- 核心行为规则可执行且结果可观察。
- 新细胞生成时，默认优先尝试父细胞朝向方向。
- 复制指令支持概率失败，失败时不会生成子细胞且流程可继续。
- 控制台支持历史指令选中复制；输入框支持 `Left/Right` 光标移动与插入编辑。
- 支持在运行中执行存档与读档指令。
- 关键交互失败时有可见提示与日志记录。
- 文档边界清晰：PRD 讲高层、系统文档讲规则、IPP 讲任务。

## 8. 全局约束

- 文档优先：改代码前先更新 PRD / 系统设计文档 / IPP。
- 系统规则只在一个主文档中完整定义，避免重复定义导致冲突。
- 不确定、冲突、缺失内容统一标记 `待确认`，不得脑补补完。

## 11. 新增目标（SEED 引导文档）

- 新增 `SEED.md`：整理当前 `AGENTS.md` 约束与项目方向（如“生态箱模拟/游戏化交互”）的高层说明。
- `SEED.md` 目标：让一个全新的智能体拿到该文件后，能够按任务自动生成可执行的 AGENT 规则草案。
- `SEED.md` 必须包含“未知项处理协议”：遇到不明确内容时，先向用户提问再继续生成。
- `SEED.md` 需写入其独立 GitHub 仓库地址（作为标准来源）。
- 需给出“将 `SEED.md` 上传到独立仓库”的多种可选方案，便于后续复用。
- 需给出“独立仓库命名方案”供用户选择（至少 3 个名称候选）。
- 独立仓库命名已确定为：`Agent_Seed`。
- `SEED.md` 与 `AGENTS.md` 需定义同步约束：规则变更需双向同步；出现版本冲突时必须先询问用户。
- `Agent_Seed` 仓库中的 `SEED.md` 应支持多项目类型（如游戏、后端服务、脚本工具等）的 AGENT 生成规则。
- 新智能体使用 `SEED.md` 时，必须先询问“当前项目类型”再选择对应模板生成 `AGENTS.md`。
- `Agent_Seed` 首版结构为：`SEED.md` + `templates/` + `examples/` + `CHANGELOG.md`。
- 指令优先级固定为：`带 ! 的用户命令 > SEED > AGENTS > 不带 ! 的用户命令 > PRD/IPP`。
- 回流机制固定为：本地 `AGENTS.md` 每次变更即提案；仅 `[GLOBAL]` 标记条目允许回流到 `Agent_Seed`。
- 仅涉及 `.md` 文件的修改流程无需额外确认，可直接执行并同步相关文档。

## 12. 新增目标（架构示例文档）

- 提供一个可复用的“具体架构设计案例”，用于解释如何把治理层、项目层、产品层分离。
- 该案例应直接映射当前项目（`SEED` / `AGENTS` / `PRD+系统文档+IPP`）并给出数据流、冲突处理与验收方式。
- 产出文档应可被新智能体直接消费，用于快速建立统一架构心智模型。
