# 系统设计：规则冲突与边界情况（Rules & Edge Cases）

## 1. 文档目的

集中记录系统规则优先级、边界场景与异常处理口径。  
本文件不定义产品高层目标，不负责任务排期。

## 2. 范围

- 指令/输入冲突优先级
- 运行时异常处理
- 文档冲突处理流程

## 3. 相关对象 / 变量

- 输入框状态（空/非空）
- 快捷键与命令解析器
- 存档读写路径与日志路径
- 错误日志记录器
- 文档集合：PRD、系统设计、IPP、AGENTS

## 4. 系统规则

- 触发条件：非法输入、资源缺失、存档异常、日志异常、文档口径冲突。
- 处理规则：
  - 命令参数非法时阻断执行并提示。
  - 快捷键优先级遵循“空输入触发控制，非空输入用于编辑”。
  - 读档失败、日志失败、浮窗失败时记录错误并保持主循环可用。
  - 文档冲突统一标记 `待确认`，禁止边实现边猜测。
- 状态变化：
  - 错误态信息进入面板与日志。
  - 必要时回退到安全默认状态（如输入框为空）。
- 输出反馈：
  - 用户可见错误提示
  - 稳定的日志记录
- 例外情况：
  - 历史规则迁移期允许存在冲突项，但必须在本文件第7节登记。

## 5. 输入 / 输出

- 玩家输入：
  - 非法命令、缺失参数、冲突快捷键
- 系统输入：
  - 存档文件、日志文件、配置值
- 系统输出：
  - 错误提示、降级行为、日志条目
- 可视化反馈：
  - 面板错误消息、不中断的主界面交互

## 6. 关键流程

1. 检测输入或运行异常。  
2. 根据优先级规则判断可恢复与不可恢复路径。  
3. 反馈提示并记录日志。  
4. 若规则冲突则标 `待确认` 并阻止直接实现。
