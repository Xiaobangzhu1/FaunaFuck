# 系统设计：可视化与调试（Visualization & Debug）

## 1. 文档目的

定义用户可见界面、可视化反馈与调试/日志入口。  
本文件不定义底层细胞行为算法，不承载开发排期。

## 2. 范围

- 左图右栏布局与展示分区
- 颜色反馈与状态可视化
- 输入交互、浮窗、日志调取与调试提示

## 3. 相关对象 / 变量

- 显示参数：`DispConfig.scale`、`DispConfig.fps`
- 布局区域：地图区、tick 区、交互区
- 面板输入状态与消息缓冲
- 日志输出：`logs/fauna.log`、`logs/recent_5m.log`
- 颜色主题：Forest Console

## 4. 系统规则

- 触发条件：每帧渲染、命令输入、快捷键事件、日志调取请求。
- 处理规则：
  - 维持左侧地图 + 顶部 tick + 右侧交互区布局。
  - 输入框为空时优先响应控制类快捷键。
  - 输入框非空时支持 `Left/Right` 移动插入点，不触发帧控制。
  - 控制台支持历史指令选中与复制。
  - DNA 浮窗支持命令、快捷键、按钮三入口及关闭动作。
  - 细胞颜色按状态与年龄分档展示。
  - 当前规则下地图不渲染方向箭头/三角形。
- 状态变化：
  - 面板消息、输入内容、浮窗状态、日志摘要可变化。
- 输出反馈：
  - 视觉层（地图/面板/浮窗）
  - 文本层（提示、错误、摘要）
- 例外情况：
  - 日志读取失败时应提示并不中断主循环。
  - 浮窗打开/关闭/拖动失败时应记录日志并保留主界面可交互。

## 5. 输入 / 输出

- 玩家输入：
  - 命令行输入
  - 快捷键
  - 按钮点击
- 系统输入：
  - 世界状态、日志文件、配置参数
- 系统输出：
  - 渲染帧
  - 状态提示
  - 日志文件与摘要
- 可视化反馈：
  - 细胞颜色变化
  - 面板文本变化
  - 浮窗开关状态

## 6. 关键流程

1. 收集输入事件。  
2. 处理命令与快捷键优先级。  
3. 更新面板/浮窗/日志状态。  
4. 渲染地图、tick 与交互区。
