# 系统设计：核心循环（Core Loop）

## 1. 文档目的

说明系统每个运行阶段如何推进，以及用户输入如何影响运行状态。  
本文件不定义 DNA/RNA 指令细节，不定义任务排期。

## 2. 范围

- 时间推进与帧循环
- 暂停/单步/回退/restart 的运行控制
- 命令输入到系统反馈的闭环

## 3. 相关对象 / 变量

- `World.paused`
- `World.ticks`
- 历史快照（帧回退能力）
- 控制命令：`pause/resume/toggle/step/back/restart/spawn/set/help/dna/log5/undo/copy/paste/save/load`
- 输入框状态（空输入时快捷键优先）
- 控制台历史指令列表与输入框光标插入点

## 4. 系统规则

- 触发条件：用户输入命令或快捷键。
- 处理规则：在主循环中读取输入，更新世界状态或返回错误提示。
- 处理规则：在输入模式下支持历史指令选中复制与光标插入点移动。
- 状态变化：`paused`、`ticks`、快照索引、细胞集合与日志状态可能变化。
- 输出反馈：面板消息、日志记录、地图渲染结果。
- 例外情况：
  - 历史帧不足时禁止回退并提示。
  - restart 异常时提示并记录日志。
  - 实时日志调出协议仍有 `待确认` 项。
  - 输入框为空时 `Left/Right` 仍用于帧控制，不用于光标移动。

## 5. 输入 / 输出

- 玩家输入：
  - 文本命令
  - 快捷键（仅输入框为空时触发控制类快捷键）
  - 历史命令选中与复制操作
- 系统输入：
  - 当前世界状态、配置参数、日志文件
- 系统输出：
  - 新帧状态
  - 面板提示
  - 日志记录
- 可视化反馈：
  - 左侧地图与顶部 tick 区域
  - 右侧交互区状态与消息

## 6. 关键流程

1. 读取输入事件（键盘/命令）。  
2. 判断输入框状态并做快捷键优先级分流。  
3. 执行命令或控制动作。  
4. 更新世界状态与快照。  
5. 刷新渲染并输出提示/日志。  

## 7. 待确认问题

- “运行中实时调出日志”是新增独立命令还是增强 `log5`，尚未统一。
- 启动默认输入 `spawn` 的展示时机与清空策略（首次执行后是否保留）待确认。
