# 系统设计：实体与状态（Entities & State）

## 1. 文档目的

定义系统涉及的主要实体、状态字段与配置项。  
本文件不定义开发任务顺序，也不承载里程碑信息。

## 2. 范围

- 世界、细胞、NTs、配置对象
- 状态字段与约束关系
- 生命周期关键状态

## 3. 相关对象 / 变量

- 实体：
  - `World`
  - `Cell`
  - `NTs`
- 细胞关键字段：
  - `x/y`
  - `gene_DNA`
  - `gene_RNA`
  - `channel`
  - `ribosome`
  - `dead`
  - `locked`
  - `age_ticks`
  - `direction`
- 关键配置：
  - `MapConfig`（尺寸、通道、指令上限）
  - `CellConfig`（初始数量、突变率、死亡阈值等）
  - `DispConfig`（fps、scale）
  - `RandomConfig`（seed）
  - `SaveConfig`（存档读写）

## 4. 系统规则

- 触发条件：初始化、每帧更新、命令执行、存档读写。
- 处理规则：
  - 世界维护细胞集合、快照历史、tick。
  - 细胞按 RNA 指令与规则变化状态。
  - 配置参数影响行为阈值与运行表现。
- 状态变化：
  - 细胞可从存活进入死亡状态。
  - 世界可在暂停/运行间切换。
  - restart 后状态重置。
- 输出反馈：
  - 渲染层可见状态变化
  - 日志记录关键操作
- 例外情况：
  - 方向字段保留但方向可视化当前不渲染。
  - 与方向可视化相关历史口径统一为 `待确认`（若与其他文档冲突）。

## 5. 输入 / 输出

- 玩家输入：
  - `spawn/set/restart/...` 等命令修改状态
- 系统输入：
  - 配置常量、随机种子、存档数据
- 系统输出：
  - 更新后的状态结构（细胞、NTs、tick、快照）
- 可视化反馈：
  - 颜色、位置、面板文本和统计信息

## 6. 关键流程

1. 初始化世界与配置。  
2. 加载（或跳过）存档。  
3. 每帧驱动细胞行为并更新状态映射。  
4. 根据输入进行状态修改。  
5. 写入日志并输出渲染。
